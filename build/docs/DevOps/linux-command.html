    # 查看系统性能占用

    # top命令
top 

P：按CPU使用率排序。

M：按内存使用率排序。

E: 切换顶部内存单位

e: 切换列表内存单位

N：按进程ID排序。

q：退出top。

k：杀死进程。

    # CentOS创建访客用户

    - 创建新用户:
sudo useradd guest

- 设置用户密码:
sudo passwd guest

- 配置sudo权限,为新用户添加到wheel组（CentOS默认的sudo权限组）
# 这行配置允许guest用户执行任何命令，并且在执行sudo时不需要输入root密码(每次登录第一次执行sudo需要输入自己的密码)。guest用户具有安装软件的权限。
sudo usermod -aG wheel guest

    # CentOS允许通过SSH方式登录

    # 打开配置文件进行编辑  
sudo vim /etc/ssh/sshd_config  

# 找到 AllowUsers 指令（如果存在），并添加 guest  
# 如果没有 AllowUsers 指令，你可能需要添加它，但请注意这可能会覆盖之前的设置  
AllowUsers root guest  

# 保存并关闭文件  
# 重启 SSH 服务使更改生效  
sudo systemctl restart sshd

    # 高版本linux授权问题

    对于centos高版本，通过执行 sudo usermod -aG wheel username 用户可能任然无法使用sudo命令，此时：

    # 执行visudo，visudo会自动打开文件/etc/sudoers并检查语法
visudo

# 找到
root ALL=(ALL:ALL) ALL

# 添加
username ALL=(ALL:ALL) ALL

    # 修改ubuntu镜像源

    # 修改成阿里云镜像源

    # 备份镜像源源文件
sudo cp  /etc/apt/sources.list /etc/apt/sources.list.bak

# 编辑镜像源源文件
sudo vim /etc/apt/sources.list

# 添加以下内容，以Ubuntu-22.04为例
